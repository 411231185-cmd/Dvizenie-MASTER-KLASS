## КАК ИСПОЛЬЗОВАТЬ

### Для одного репо:

1. Откройте репо в VS Code
2. Скопируйте промпт выше
3. Замените `{REPO_NAME}` на название репо
4. Вставьте в ChatGPT/Claude
5. Получите анализ и рекомендации
6. Создайте CHECKLIST.md на основе результата

### Для всех 13 репо:

1. Подготовьте список всех 13 репо
2. Откройте их все в VS Code (File → Open Folder...)
3. Для каждого повторите процесс выше
4. Создайте персональные чек-листы

## ПРИМЕРЫ АНАЛИЗОВ

### Пример 1: Python проект

```
═══════════════════════════════════════════
📋 АНАЛИЗ: AI-Bot-Project
═══════════════════════════════════════════

✅ ТЕКУЩЕЕ СОСТОЯНИЕ
───────────────────
- Python 3.9 проект для Telegram бота
- 15 модулей, 2000+ строк кода
- Документация на 80% готова
- Нет unit-тестов

❌ КРИТИЧНЫЕ ЗАДАЧИ
───────────────────
1. Исправить 8 неиспользуемых импортов
2. Добавить обработку ошибок в main.py
3. Обновить requirements.txt версии пакетов
4. Пустой except блок в bot_handler.py

🔴 КРИТИЧНЫЕ (1-2 дня):
  1. Удалить неиспользуемые импорты
  2. Обновить зависимости
  3. Добавить ошибки обработку

🟠 ВЫСОКИЕ (неделя):
  1. Написать unit-тесты (минимум 50%)
  2. Настроить CI/CD GitHub Actions
  3. Добавить logging

🟡 СРЕДНИЕ (месяц):
  1. Рефакторинг структуры проекта
  2. Добавить type hints везде
  3. Оптимизировать database запросы
```

## ВАРИАНТЫ ДЛЯ РАЗНЫХ ТИПОВ ПРОЕКТОВ

### 📱 Frontend проект (React/Vue/Angular)

```
СПЕЦИФИЧНЫЕ ВОПРОСЫ:
- Какие компоненты требуют рефакторинга?
- Есть ли проблемы с производительностью?
- Нужна ли оптимизация bundle size?
- Какой state management используется?
- Нужны ли unit/E2E тесты?
```

### 🔐 Backend проект (API/Server)

```
СПЕЦИФИЧНЫЕ ВОПРОСЫ:
- Есть ли проблемы с security?
- Какие endpoints требуют оптимизации?
- Нужна ли документация API (Swagger)?
- Есть ли проблемы с database?
- Нужна ли миграция данных?
```

### 🤖 Machine Learning проект

```
СПЕЦИФИЧНЫЕ ВОПРОСЫ:
- На каком этапе разработки модель?
- Какие метрики качества отслеживаются?
- Нужна ли оптимизация модели?
- Требуется ли переобучение?
- Есть ли pipeline для обработки данных?
```

### 📦 Library/Package проект

```
СПЕЦИФИЧНЫЕ ВОПРОСЫ:
- Есть ли документация для пользователей?
- Нужна ли оптимизация производительности?
- Какова версионность (semver)?
- Требуется ли обновление в package manager?
- Есть ли примеры использования?
```

## СОХРАНЕНИЕ РЕЗУЛЬТАТОВ

После получения анализа:

1. **Создайте CHECKLIST.md** в корне репо
2. **Скопируйте рекомендации** из анализа
3. **Структурируйте задачи** по приоритетам
4. **Добавьте в Git** и коммитьте

```bash
git add CHECKLIST.md
git commit -m "docs: add project checklist with analysis"
git push
```
